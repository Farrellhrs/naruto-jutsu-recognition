\chapter{Gameplay}

\section{Game Modes}
The app supports four user modes:
\begin{itemize}[leftmargin=2em]
\item Battle: defend and counterattack against Sasuke with HP and chakra systems.
\item Free: trigger any supported jutsu without fixed target constraints.
\item Speed: complete a selected jutsu sequence as fast as possible.
\item Tutorial: follow guided signs for a selected target jutsu.
\end{itemize}

\section{Battle Mechanics}
Battle mode adds a state machine around CV inference:
\begin{enumerate}[leftmargin=2em]
\item Enemy attack phase selects an incoming jutsu.
\item Player defend phase expects a specific counter jutsu sequence.
\item Player attack phase allows offensive jutsu launches.
\item Round end restores partial chakra and advances battle pacing.
\end{enumerate}

Counter mapping examples:
\begin{itemize}[leftmargin=2em]
\item Fireball or Burning Ash is countered by Water Dragon.
\item Lightning is countered by Rasengan.
\end{itemize}

\section{Visual and Audio Feedback}
\begin{itemize}[leftmargin=2em]
\item Overlay guidance shows current target sign and sequence progression.
\item Particle and scene effects represent each jutsu style.
\item Audio loops adapt volume and rate for active effects such as Rasengan and Wind.
\end{itemize}
