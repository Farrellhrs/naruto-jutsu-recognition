\chapter{Dataset}

\section{Source and Annotation Format}
The training dataset is based on YOLO-style labels where each annotation stores class index and normalized bounding box coordinates.

Primary dataset configuration:
\begin{itemize}[leftmargin=2em]
\item Location: \texttt{\detokenize{hand_gesture_model/naruto-hand-sign-4/}}
\item Config file: \texttt{\detokenize{hand_gesture_model/naruto-hand-sign-4/data.yaml}}
\item Number of classes: 12
\item Class names: bird, boar, dog, dragon, hare, horse, monkey, ox, ram, rat, snake, tiger
\end{itemize}

\section{Data Splits}
The dataset uses train, validation, and test folder splits:
\begin{itemize}[leftmargin=2em]
\item \texttt{\detokenize{train/images}} and \texttt{\detokenize{train/labels}}
\item \texttt{\detokenize{valid/images}} and \texttt{\detokenize{valid/labels}}
\item \texttt{\detokenize{test/images}} and \texttt{\detokenize{test/labels}}
\end{itemize}

\section{Label Alias Note}
In iOS gameplay logic, \textit{hare} is normalized to \textit{rabbit} for asset naming and sequence consistency. This alias handling should be kept explicit in deployment documentation.
